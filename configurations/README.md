# Configurations

A Deployment Profile shows what it looks like to run a domain as a Plektis node. It sets out what the node mounts, what it runs on, where it has to stop, and what it holds onto.

What a profile mounts can be public or private. Some domains rest on open knowledge and mount a public skill. Others turn on scored instruments that stay in a Layer 4 or 5 deployment. Either way, the profile is the wiring and points at where the expertise plugs in.

A Deployment Profile is public. It carries the shared vocabulary of the domain and the shape of the node that runs it, and nothing that would reconstruct a proprietary instrument. See Section 6.5 (Application Mounting) for how a profile maps to the mounting interface.
